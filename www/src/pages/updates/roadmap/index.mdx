    .filter(item => item.status !== '')
    .filter(item => item.status !== 'complete');
export const compoonetUsageComplete = compoonetUsageDocumentation.filter(
    item => item.status === 'complete',
);
export const compoonetUsageToDo = compoonetUsageDocumentation.filter(item => item.status === '');

export const processPill = status => {
    if (status == 'complete') {
        return <Pill color="green">{status}</Pill>;
    } else if (status == 'in progress') {
        return <Pill color="yellow">{status}</Pill>;
    } else {
        return <Pill>{status || 'not started'}</Pill>;
    }
};

## What's happening now

### Component usage documentation

This exercise is an effort to provide clear documentation on use cases, anatomy, specs, accessibility needs, and how to use a particular component. In some cases, some usage documentation may already exist but will be updated due to this effort.

Updating components will occur in order of usage patterns. The more popular the component is in adoption, the sooner the component will be addressed.

This project is expected to carry over from Q2 into mid-H2 2023.

## What's next

### Updated Toast component

The [Toast component](/components/toast/ios/) is currently only supported by our native platforms, iOS and Android. This effort will refresh Toast's current version and support the Web platform (Mobile and Desktop).

Work for this project is expected to begin in Q2 or Q3 2023.

### Semantic color tokens

This work will introduce a more semantic approach to provide an intentional methodology for describing colors and how they should be applied by adding a second tier of tokens (semantic) that references our current token color construct (base).
